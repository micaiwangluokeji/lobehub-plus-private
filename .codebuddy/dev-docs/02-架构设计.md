# LobeHub 架构设计

> **来源**：https://lobehub.com/zh/docs/development/basic/architecture
> **所属**：开发 > 基础

## 1. 应用架构概览

| 层 | 描述 |
| :--- | :--- |
| **前端** | Next.js RSC + React Router DOM 混合路由 SPA |
| **后端 API** | RESTful WebAPI + tRPC Routers |
| **运行时** | Model Runtime + Agent Runtime |
| **认证** | Better Auth（邮箱密码 + SSO） |
| **数据存储** | PostgreSQL + Redis + S3 |
| **市场** | Agent 市场 + MCP 工具市场 |

## 2. 前端架构

### 路由方案

采用 **Next.js RSC + React Router DOM 混合路由**：

- **Next.js App Router**: 处理认证页、SSR、静态路由，代码位于 `src/app/(backend)/`。
- **React Router DOM**: 承载主聊天 SPA、Agent 界面，代码位于 `src/spa/` + `src/routes/`。

### 主要技术栈

- UI 组件: `@lobehub/ui`、antd
- CSS-in-JS: antd-style
- 状态管理: zustand (slice 模式)
- 数据请求: SWR + tRPC
- 国际化: react-i18next

### 组件使用优先级

项目组件 (`src/components/`) → `@lobehub/ui` → Ant Design (`antd`)

## 3. 后端 API

两种 API 形式：

- **RESTful WebAPI** (`src/app/(backend)/webapi/`): 处理 chat 流式响应、TTS、文件服务等特殊端点。
- **tRPC Routers** (`src/server/routers/`): 类型安全的主要业务路由，按运行时分组：
  - `lambda/`: 主业务（agent, session, message, topic, file, knowledge, settings 等）
  - `async/`: 耗时异步操作
  - `tools/`: 工具调用（search, MCP, market）
  - `mobile/`: 移动端专用

## 4. Runtime

### Model Runtime (`@lobechat/model-runtime`)

LLM API 适配层，抹平 30+ 不同 AI Provider 的 API 差异。无状态，每次调用独立。

### Agent Runtime (`@lobechat/agent-runtime`)

Agent 编排引擎，核心功能：
- Plan-Execute 循环（核心状态机）
- 工具调用与批量执行
- Human-in-the-Loop（人工审批）
- 上下文压缩
- 用量与成本追踪
- 多 Agent 协作

## 5. 数据存储

| 存储 | 用途 |
| :--- | :--- |
| **PostgreSQL** | 主数据库 |
| **Redis** | 缓存、会话状态、速率限制 |
| **S3** | 文件存储 |

## 6. 性能优化

代码拆分、懒加载、SWR 缓存、边缘缓存、流式响应 (SSE)、虚拟滚动。

## 7. 安全措施

CSRF 防护、XSS 防护 (CSP)、速率限制、SSRF 防护、SQL 注入防护、密钥管理。

## 8. 开发和部署

- **版本控制**: Git + GitHub，gitmoji commit 规范
- **代码质量**: ESLint, Stylelint, TypeScript, 循环依赖检测, 死代码检测
- **测试**: Vitest + Cucumber/Playwright
- **CI/CD**: GitHub Actions
- **部署**: Vercel、Docker、各大云平台自托管
