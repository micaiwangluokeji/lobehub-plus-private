# LobeHub 功能开发完全指南

> **来源**：https://lobehub.com/zh/docs/development/basic/feature-development
> **所属**：开发 > 基础

## 概述

本文档通过一个具体的功能需求案例（RFC 021 - 自定义助手开场引导），详细阐述了在 LobeHub 项目中开发一个新功能所需的全部步骤和流程。

## 一、更新 Schema（数据库层）

- **位置**: `packages/database/src/schemas/`
- **工具**: Drizzle ORM
- **操作**: 在对应的数据库表（如 `agents`）中增加新的字段（如 `opening_message`, `opening_questions`）。
- **后续步骤**: 调整完 Schema 后，需要生成并执行数据库迁移文件。

## 二、更新数据模型（类型定义层）

- **位置**: `packages/types/src/`
- **原因**: LobeHub 维护了独立于 ORM 的数据模型，以便更好地服务于前端需求。
- **操作**: 在对应的接口（如 `LobeAgentConfig`）中增加新的可选字段。

## 三、Service / Model 各层实现（业务逻辑层）

后端完整的分层架构：

1. **Client Service** (`src/services/`): 前端可复用的业务逻辑，通过 tRPC 调用后端。
2. **tRPC Router** (`src/server/routers/`): 后端入口，用于校验输入并路由到 Server Service。
3. **Server Service** (`src/server/services/`): 服务端业务逻辑。
4. **Repository / DB Model** (`packages/database/`): 封装数据库操作。

## 四、前端实现（表示层）

### 1. 数据流 Store 实现 (zustand)

- **更新配置初始值**: 在 `src/const/settings/agent.ts` 的 `DEFAULT_AGENT_CONFIG` 中添加新字段的默认值。
- **添加 Selectors**: 在 Store 的 `selectors` 文件中添加用于高效访问新字段的函数。
- **使用 Action**: 通过已有的 `setAgentConfig` action 更新配置。

### 2. 国际化 (i18n)

- **位置**: `src/locales/default/` 下的 namespace 文件
- **操作**: 添加新 UI 文案的 key（英文），遵循扁平的 dot notation 命名规范。
- **翻译**: 开发时手动翻译中英文 JSON 文件，CI 会自动生成其他语言的翻译。

### 3. UI 实现

- 使用 `useStore(selectors.xxx)` 读取配置
- 使用 `useStore((s) => s.setAgentConfig)` 更新配置
- 使用 `useTranslation('namespace')` 获取国际化文案

## 五、测试

- **工具**: Vitest
- **建议**: 使用 `bunx vitest run --silent='passed-only' '[file-path]'` 运行单文件测试
- 新增可选字段一般不会破坏现有测试
