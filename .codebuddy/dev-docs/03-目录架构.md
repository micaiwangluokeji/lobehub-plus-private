# LobeHub 目录架构

> **来源**：https://lobehub.com/zh/docs/development/basic/folder-structure
> **所属**：开发 > 基础

## 1. 整体架构: Monorepo

```
lobehub/
├── apps/            # 独立应用（desktop/ 等）
├── packages/         # 共享包（@lobechat/*）
├── src/              # 主应用源码（核心开发目录）
├── locales/          # i18n 翻译文件
├── e2e/              # E2E 测试
└── docs/             # 文档
```

### `packages/` 目录

- `agent-runtime/`: Agent 运行时
- `database/`: 数据库 schemas、models、repositories
- `model-runtime/`: 模型运行时（各 AI 提供商适配）
- `context-engine/`: 上下文引擎
- `conversation-flow/`: 会话流程

## 2. 核心目录: `src/`

```
src/
├── app/             # Next.js App Router（路由组和 API 路由）
├── business/        # Cloud 版专用业务逻辑
├── components/      # 可复用 UI 组件
├── config/          # 应用配置（客户端与服务端环境变量）
├── const/           # 应用常量和枚举
├── features/        # 业务功能模块（按领域划分）
├── hooks/           # 全应用复用的自定义 Hooks
├── layout/          # 全局布局组件
├── libs/            # 第三方集成
├── server/          # 服务端模块
├── services/        # 客户端服务接口
├── store/           # zustand 状态管理
├── styles/          # 全局样式
├── types/           # TypeScript 类型定义
└── utils/           # 通用工具函数
```

## 3. 路由/应用入口: `src/app/`

- `(backend)/`: 所有服务端 API 路由
- `[variants]/`: 处理不同平台和设备变体
  - `(main)/`: 主应用 SPA（React Router DOM）
  - `(desktop)/`: 桌面端专用路由
  - `(mobile)/`: 移动端专用路由
  - `onboarding/`: 新用户引导页面
  - `share/`: 分享页面

## 4. 数据流架构

> **用户交互** → **React UI** → **Store Actions** (zustand) → **Client Service** (tRPC) → **tRPC Router** → **Server Service** → **DB Model** (Drizzle ORM) → **PostgreSQL**

## 5. SPA 路由架构

桌面端 SPA 路由（定义于 `desktopRouter.config.tsx`）：

- `/`: 首页
- `/agent/:aid`: Agent 会话
- `/settings/:tab`: 设置页面
- `/image`: 图像生成
- `/memory`: 记忆管理
- `/community`: 社区发现

移动端 SPA 路由（定义于 `mobileRouter.config.tsx`）：

- `/`: 首页
- `/agent/:aid`: Agent 会话
- `/settings/:tab`: 设置页面
- `/me`: 个人中心
- `/share/t/:id`: 分享话题
