## 本地开发环境调试（7月2日 11:00-13:10）

### 关键发现

**1. Next.js 16 中间件约定**
- `src/proxy.ts` 就是中间件文件（Next.js 16 弃用了 `middleware.ts`，改用 `proxy.ts`）
- 同时存在 `src/middleware.ts` 和 `src/proxy.ts` 会冲突报错

**2. 本地开发模式**
- `bun run dev:spa`（9876）：仅主 SPA，无认证页面，适合纯 UI 开发
- `bun run dev`（全栈 3010+9876）：有完整后端，认证页面可用
- 访问地址：`localhost:9876/admin`（Vite SPA），API 通过 Vite proxy 到 localhost:3010
- 不需要 Debug Proxy（`app.lobehub.com/...`）

**3. OpenAPI 认证方式**
- `/api/v1/*` 使用 API Key（Bearer token）或 OIDC，**不支持 session cookie**
- 开发 bypass：请求头 `lobe-auth-dev-backend-api: 1` + 环境变量 `MOCK_DEV_USER_ID`
- tRPC 路由使用 Better Auth session cookie

**4. API 返回格式**
- 列表接口：`{ data: { <items>: [...], total: N }, message, success, timestamp }`
- items key 依端点而异：`users`、`roles`、`permissions`
- 用户角色接口（`/api/v1/users/:id/roles`）：`data` 直接是数组

**5. 数据库**
- PostgreSQL 跑在 Docker `lobe-postgres` 容器，密码 `lobe_dev_2024`
- Seed 脚本硬编码邮箱 `903164524@qq.com` 授予 super_admin
